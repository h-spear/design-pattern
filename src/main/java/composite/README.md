# Composite(컴포지트)

-   개별 객체와 복합 객체를 똑같이 취급하는 것
-   객체들을 트리 구조로 구성하여 부분과 전체를 나타내는 계층 구조로 만들 수 있다.
    -   Leaf : 단일 객체, 내부에 다른 것을 넣을 수 없다
    -   Composite : 복합체, Leaf나 Composite를 넣을 수 있다.
    -   Component : Leaf, Composite의 역할을 동일시하며, 둘의 상위 클래스이다.
    -   디렉토리-파일 구조와 비슷
-   재귀적으로 구현
-   객체들이 모두 같은 타입으로 취급되기 때문에 새로운 클래스 추가가 용이하다.
-   단일 객체 및 집합 객체를 구분하지 않고 코드 작성이 가능하여 사용자 코드가 단순해진다.
-   복잡한 트리 구조를 편리하게 사용할 수 있다(재귀)
-   설계가 지나치게 범용성을 가지게 되고, 복합체의 구성 요소에 제약을 가하기 어려운 구조가 된다.
-   기능이 많이 다른 클래스들의 공통 인터페이스를 제공하는 것이 어려울 수 있다.
    -   지나치게 일반화해야 하기 때문에 이해하기 어려운 코드가 될 수 있다.
